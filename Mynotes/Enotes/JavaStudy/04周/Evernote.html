  • 每日学习路线：

  • 单个集合学习路线：

  • 1.Collection 集合接口
    • 1.1 List (是Collection的子接口，添加了许多对于下标进行操作的方法)
      • 特点：有序的，且可重复的
        • ArrayList：
          • 数据结构：一维数组
        • LinkedList：(模拟栈和队列模式)
          • 数据结构：双向链表
      • ArrayList vs LinkedList  
        • 添加到最后一个：都一样快
        • 插入：LinkedList快
        • 删除最后一个：一样快
        • 删除中间：LinkedList快
        • 查询：ArrayList快
    • ps：工作中使用ArrayList更加频繁，因为查询的次数是最多的。如果使用到栈和队列模式，就使用LinkedLis
    • Vector：
      • 数据结构：一维数组
    • ArrayList vs Vector
      • ArrayList：线程不安全
      • Vector：线程安全的
    • Stack（继承Vector）：
      • 数据结构：一维数组（模拟栈的模式）
    • 1.2 Set (是Collection的子接口)
      • 特点：无序的，且不可重复的
      • ps：
        • 1.正确理解无序
        • 无序：存入顺序与取出顺序不一致，无序不代表随机 
    • HashSet： 
      • 数据结构：Hash表（Hash数组）
      • 应用场景：单个元素的去重
    • TreeSet：
      • 数据结构：二叉树
      • 特点：自然排序
  • 2.Map 接口 -- 映射关系k-v
    • 特点：键值对存储
      • HashMap：
        • 数据结构：Hash表（Hash数组）
        • 特点：键的唯一性 --- 键不可以重复，值可以重复的
        • 应用场景：key元素的去重，又能加入value值
      • HashTable：
        • 数据结构：Hash表（Hash数组）
        • 应用场景：key元素的去重，又能加入value值
      • ConcurrentHashMap：
        • 数据结构：Hash表（Hash数组）
        • 应用场景：key元素的去重，又能加入value值
    • HashMap vs HashTable vs ConcurrentHashMap
      • 存储null值：
        • HashMap : 可以
        • HashTable : 不可以
        • ConcurrentHashMap：不可以
    • 线程安全：
      • HashMap : 不安全
      • HashTable : 安全
      • ConcurrentHashMap：安全
      • HashTable:线程安全，直接在方法上加锁，效率低
      • ConcurrentHashMap:在存储过程中加锁，分段式加锁，效率高
      • TreeMap：
        • 数据结构：二叉树
        • 特点：排序（仅仅对key排序）
    • ps：
      • 1.Collection提供了获取迭代器的方法，拥有了遍历集合的功能
      • 2.Map没有提供获取迭代器的方法，就不能直接遍历集合，可以有其他方法来遍历集合
  • 面试题：
    • 1.Collection 和 Map 集合的区别？
    • Collection 存储单个对象
    • Map 存储两个对象

class CopyThread extends Thread{
         private boolean isFinished = false;
         private long s;
         private long e;
         private File file;
         private File targetfile;
         public CopyThread() {}
         public CopyThread(long s, long e, File file,File targetfile) {
                 this.s = s;
                 this.e = e;
                 this.file = file;
                 this.targetfile = targetfile;
         }
         public boolean getIsFinished() {
                 return isFinished;
         }
         @Override
         public void run() {
                 try {
                          RandomAccessFile raf = new RandomAccessFile(file, "r");
                          RandomAccessFile wraf = new RandomAccessFile(targetfile, "rw");
                          raf.seek(s);
                          wraf.seek(s);
                          //System.out.println(s +" " + raf.getFilePointer());
                          byte[] buf = new byte[1024];
                          int len = -1;
                          while ((len = raf.read(buf))!=-1) {
                                  wraf.write(buf, 0, len);
                                  //获取当前文件指针的方法getFilePointer
                                  if (e - raf.getFilePointer()<1024) {
                                           break;
                                  }
                          }
                          //读取剩余部分
                          if (raf.getFilePointer()!=e) {
                                  byte[] bufleft = new byte[(int)(e-raf.getFilePointer())];
                                  len = raf.read(bufleft, 0, bufleft.length);
                                  wraf.write(bufleft, 0, bufleft.length);
                          }
                          raf.close();
                          wraf.close();
                 } catch (IOException e) {
                          e.printStackTrace();
                 }
                 //表示该线程已经完成
                 isFinished = true;
                 System.out.println(Thread.currentThread().getName()+"已经完成...");
         }
}
